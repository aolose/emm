# EMM 博客性能优化记录

## 一、请求层优化

### 1.1 消除公开页面的冗余 API 请求

**问题**：`+layout.ts` 每次客户端导航（切文章、打开关于、翻标签）都调用 `/api/statue`。这个接口返回 `statue`（登录态）、`sys`（系统状态）、`title/desc/key`（SEO 配置）、`pwdSalt`（密码盐值），但公开页面浏览只需要其中零个字段——鉴权只对 `/admin` 有意义，站点配置首屏加载后不再变化。

**方案**：将 `apiLoad` 替换为自定义 `load` 函数，客户端公开页面导航时直接返回缓存数据，零网络请求。

```ts
// +layout.ts
if (browser && loaded && !path.startsWith('/admin')) {
    return { p: params, d: cacheData };
}
```

**效果**：切文章、翻页、浏览公开内容不再触发 `/api/statue`。仅访问 `/admin` 路径时才重新请求做鉴权校验。

### 1.2 为 `__data.json` 添加 ETag 支持

**问题**：SSR 渲染的 HTML 页面有 SvelteKit 自动生成的 ETag（支持 304 Not Modified），但 `__data.json`（SPA 导航数据）返回 `Cache-Control: private, no-store`，每次都全量传输。

**方案**：在所有公开页面的 `load` 函数中通过 `event.setHeaders({ etag })` 设置 ETag，利用 SvelteKit 内置的 304 机制。

涉及页面：首页、文章页、文章列表、标签列表、标签筛选、关于页，共计 6 个。

```ts
// 以文章页为例 (+page.server.ts)
const etag = `"${Bun.hash(JSON.stringify(d)).toString(36)}"`;
event.setHeaders({ etag });
```

通用 `+page.ts` 使用 portable hash 替代 `Bun.hash`（客户端不可用）。

**效果**：浏览器刷新页面时，数据未变则返回 304 不传输 body。

### 1.3 开启 SvelteKit 链接预加载

**问题**：用户 hover 链接 → 点击 → 等待 `load` 函数执行，中间有可感知的延迟。

**方案**：`<body>` 添加 `data-sveltekit-preload-data="hover"`，SvelteKit 客户端路由在 hover 时后台静默请求目标页面的 `load` 数据。点击时数据已就绪，导航秒开。

```html
<body data-sveltekit-preload-data="hover">
```

**额外修复**：上一篇/下一篇链接原来使用 `<button onclick={goto(...)}>`，SvelteKit 无法识别为导航链接。改为原生 `<a href>` 标签后自动享受预加载。

### 1.4 评论组件懒加载

**问题**：文章页 `onMount` 时立即请求 `/api/cmLs`，即使用户从不滚动到评论区。

**方案**：用 `IntersectionObserver` 替换 `onMount` 中的 `go()` 调用，评论区域距离视口 200px 时才发起请求。`slug` 变化（切文章）时自动重置 observer。

```ts
$effect(() => {
    const _ = slug;
    loaded = false;
    ls = [];
    if (!el) return;
    const observer = new IntersectionObserver(
        (entries) => {
            if (entries[0].isIntersecting) { loaded = true; go(); observer.disconnect(); }
        },
        { rootMargin: '200px' }
    );
    observer.observe(el);
    return () => observer.disconnect();
});
```

**效果**：文章页打开时不再请求评论 API，用户滚动到评论区附近才开始加载。

## 二、图片加载优化

### 2.1 CSS background-image → 原生 `<img>`

**问题**：文章 banner 图使用 CSS `background-image`（inline style），浏览器预加载扫描器无法发现（需等 CSSOM 构建），大图同步解码阻塞主线程导致渲染卡顿。

**方案**：替换为原生 `<img>` 标签 + `object-fit: cover` + `decoding="async"` + `fetchpriority="high"`。

```html
<img class="bg-img" src={currentBg} alt="" decoding="async" fetchpriority="high" />
```

```css
.bg-img {
    position: absolute; inset: 0;
    width: 100%; height: 100%;
    object-fit: cover;
    object-position: center;
}
```

**关键差异 vs 常见建议**：
- `fetchpriority="high"` 而非 `loading="lazy"` — hero banner 应尽快加载
- `decoding="async"` — 异步解码不阻塞主线程，消除逐块渲染的卡顿感
- 默认图也改用 Vite import，享受 hash 和构建优化

### 2.2 Banner 渐变切换

**问题**：切文章时 banner 图硬切，闪烁突兀。

**方案**：双 `<img>` + `{#key}` + `onload` 门控的 opacity 动画。

流程：
1. `targetSrc` 变化 → 旧图存入 `prevBg`（垫底，`opacity: 1`）
2. `{#key currentBg}` 强制销毁重建新 `<img>` 元素
3. 新图默认 `opacity: 0`，`onload` 触发后 `opacity: 0 → 1` 淡入动画
4. 500ms 后移除旧图 DOM

```css
.bg-img { opacity: 0; }
.bg-img.loaded { animation: bgFadeIn 0.4s ease-in-out forwards; }
.bg-old { opacity: 1; z-index: 1; }
@keyframes bgFadeIn { from { opacity: 0; } to { opacity: 1; } }
```

**踩坑**：不加 `{#key}` 时 Svelte 复用了同一个 `<img>` 只改 `src`，动画失效。不加 `onload` 门控时，动画在图片下载期间播放，等于没效果。

## 三、用户体验优化

### 3.1 导航加载指示器

**问题**：无网络状态下的导航没有任何视觉反馈。

**方案**：CSS-only 顶部扫光条，零依赖。

```svelte
{#if $navigating}
    <div class="nav-indicator" transition:fade={{ duration: 150 }}></div>
{/if}
```

```css
.nav-indicator {
    position: fixed; top: 0; height: 2px;
    background: linear-gradient(90deg, transparent, #3b82f6 20%, #60a5fa 80%, transparent);
    animation: nav-load 1.5s ease-in-out infinite;
}
@keyframes nav-load {
    0% { width: 0; left: 0; }
    50% { width: 50%; }
    100% { width: 0; left: 100%; }
}
```

**效果**：导航时顶部出现两端透明中间蓝色的扫光条，`$navigating` 变 false 时 150ms 淡出消失。加载极快时（预加载命中）几乎不闪现。

### 3.2 图标字体 FOIT 修复

**问题**：图标字体 `icon.woff2` 加载期间，所有 `.icon` 元素不可见（Flash of Invisible Text）。

**方案**：`@font-face` 添加 `font-display: swap`，加载期间使用系统字体回退。

```css
@font-face {
    font-family: 'icon';
    src: url('icon.woff2') format('woff2'), ...;
    font-display: swap;
}
```

## 四、Bug 修复

### 4.1 图片查看器在切文章后失效

**问题**：`imageViewer` action 在 mount 时创建 `new Viewer(node)`。SPA 导航时 DOM 节点复用，旧 Viewer.js 实例持有上一篇文章的图片引用，新文章图片点击无反应。

**方案**：action 接收 `slug` 参数，`update` 时销毁旧实例并重建。

```ts
export const imageViewer = async (node: HTMLElement, slug: string) => {
    // ...
    return {
        update(newSlug: string) {
            if (newSlug !== slug) {
                slug = newSlug;
                clearTimeout(timeoutId);
                timeoutId = setTimeout(init, 600);
            }
        },
        destroy() { /* clean up */ }
    };
};
```
